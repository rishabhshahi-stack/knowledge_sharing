Example A · Live demo

Read a column. Colour it green or red.

Apps Script 
function colourCodeDeltas() {
  const sheet = SpreadsheetApp.getActiveSheet();
  const range = sheet.getRange("C2:C10");
  const values = range.getValues();

  for (let i = 0; i < values.length; i++) {
    const val = values[i][0];
    const cell = range.getCell(i + 1, 1);
    // green if positive, red if not
    const colour = val > 0 ? "#d4edda" : "#f8d7da";
    cell.setBackground(colour);
  }
}
1
getActiveSheet() — no config, no file IDs. Grabs the sheet you have open right now.
2
getValues() reads the entire column in one call — returns a 2D array. Fast and simple.
3
setBackground() writes the formatting back. The same API writes values, formulas, fonts, borders — anything.
4
Wrap this in a time trigger and it runs automatically every week. This is the core pattern — read, compute, write back.
What this code does
Reads cells C2 through C10 — a column of delta values. For each row it checks whether the number is positive or negative, then paints the cell background green or red. No manual formatting, no formulas — run it once or put it on a weekly trigger and it updates in one pass.
Example B · Live demo

Call an external API. Write structured results into Sheets.

Apps Script 
function fetchMarketData() {
  const url = "https://api.frankfurter.app/latest?from=EUR&to=USD";

  const resp = UrlFetchApp.fetch(url);
  const data = JSON.parse(resp.getContentText());

  const rate    = data.rates.USD;
  const apiDate = data.date;           // date of latest source data
  const pulled  = new Date();          // timestamp of this pull

  const sheet = SpreadsheetApp.getActiveSheet();

  // Write headers
  sheet.getRange("A1").setValue("Metric");
  sheet.getRange("B1").setValue("Value");

  // Write data rows
  sheet.getRange("A2").setValue("EUR/USD Rate");
  sheet.getRange("B2").setValue(rate);

  sheet.getRange("A3").setValue("Latest data date");
  sheet.getRange("B3").setValue(apiDate);

  sheet.getRange("A4").setValue("Pulled at");
  sheet.getRange("B4").setValue(pulled.toLocaleString());
}
1
UrlFetchApp.fetch() — the single bridge between Sheets and the outside world. Any URL, any API.
2
We write headers in row 1 first, then data rows below — same structure you'd build manually in a sheet.
3
apiDate tells you when the source data was last updated. pulled is the timestamp of when this script ran — so you always know data freshness.
4
This exact pattern — fetch → parse → write labelled rows — is how the trading dashboard writes metric data from Databricks into Sheets every week.
What this code does
Calls a live exchange rate API, then writes a small structured table: the EUR/USD rate, the date of the latest available source data, and a timestamp of when the script last ran. Three rows, clearly labelled — no manual copying, always fresh.

How it is built
Backend — Code.gs: Monday trigger fires UrlFetchApp against Databricks REST API, parses JSON, writes all metric rows. Deltas computed in-script before writing.

Frontend — index.html: single-page web app served via doGet(). Reads pre-computed Sheets data via google.script.run and renders metric cards, channel tables, and time views in HTML/CSS/JS.
